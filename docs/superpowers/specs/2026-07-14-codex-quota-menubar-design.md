# Codex 剩余额度菜单栏设计

## 目标

创建一个原生 macOS 菜单栏应用，长期显示 Codex 当前剩余额度。应用不占用 Dock，只读取 Codex 已写入本机的额度记录，不联网、不修改 Codex 数据。

菜单栏格式固定为：

```text
Codex [██████░░░░] 60%
```

百分比表示剩余额度，不是已用额度。

## 数据来源与选择规则

- 扫描 `~/.codex/sessions` 下最近更新的 JSONL 会话文件。
- 从 `event_msg` / `token_count` 记录的 `rate_limits` 中提取额度窗口。
- 剩余额度按 `100 - used_percent` 计算，并限制在 `0...100`。
- 如果一条最新记录同时包含多个有效额度窗口，显示剩余百分比最低的窗口，避免隐藏更紧张的限制。
- 当前只返回一个每周额度窗口时，直接显示该窗口的剩余额度。
- 每 15 秒重新读取一次；用户也可以点击“立即刷新”。

本应用显示的是 Codex 最近一次写入本地的服务端额度快照。Codex 长时间没有产生新记录时，显示值也不会更新，菜单中必须展示快照时间，避免把旧数据误认为实时查询结果。

## 界面与交互

- 菜单栏标题使用 10 格条形图：实心格表示剩余额度，空心格表示已消耗额度。
- 百分比四舍五入为整数。
- 无有效数据时显示 `Codex [░░░░░░░░░░] --%`。
- 点击菜单栏项目后显示：
  - 最近快照时间；
  - “立即刷新”；
  - “退出”。
- 不提供设置页、颜色配置、通知、开机自启或 Dock 图标。

## 实现结构

- 使用 Swift、AppKit 和 `NSStatusItem` 实现原生菜单栏应用。
- 额度解析逻辑与菜单栏控制器分离，以便对 JSONL 选择、百分比计算和条形图渲染进行单元测试。
- 使用 `Timer` 每 15 秒触发刷新；文件扫描和解析在后台队列执行，UI 更新回到主线程。
- 读取失败不弹窗、不打断用户，只切换为无数据状态。
- 应用包使用 `LSUIElement = true` 隐藏 Dock 图标。

## 打包与交付

- 构建为 Apple Silicon 原生可执行文件，兼容当前 MacBook Air M5。
- 打包为标准 `.app` bundle，包含合法 `Info.plist`、可执行权限和 Launch Services 注册。
- 最终应用放入本次任务的 `outputs` 目录。
- 不安装到 `/Applications`，除非用户之后明确要求。

## 验证标准

1. 解析测试覆盖：单窗口、多个窗口取最低剩余值、越界百分比、无效 JSON、无额度记录。
2. 条形图测试覆盖 `0%`、`60%`、`100%` 和无数据状态。
3. Swift 编译和全部测试通过。
4. `Info.plist` 通过 `plutil -lint`。
5. macOS 将产物识别为 application bundle，且 Launch Services 注册成功。
6. 实际启动后出现菜单栏项目、不出现 Dock 图标；菜单栏百分比与最新本地额度记录一致。
7. 点击“退出”后应用进程结束，不留下后台进程。

## 明确不做

- 不调用私有 OpenAI 接口主动刷新额度。
- 不估算尚未写入本地的消耗。
- 不修改 Codex 桌面客户端界面。
- 不添加第三方运行时或菜单栏框架。
