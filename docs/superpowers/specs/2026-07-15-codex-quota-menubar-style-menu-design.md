# Codex Quota 菜单栏样式与分发设计

## 目标

在现有 Codex Quota 菜单栏 App 上增加三种 macOS 电池视觉样式、`Codex` 文字显示开关和配置持久化，并提供适合发送给同事的 ZIP 包。额度读取、15 秒刷新和“显示剩余百分比”的语义保持不变。

## 菜单栏样式

App 内置三种样式：

1. **A · 原生电池**（默认）：macOS 风格电池轮廓，内部使用连续填充，百分比显示在电池右侧。
2. **B · 数字徽章**：去掉电池外框，使用约 `34×18pt` 的圆角徽章和尽可能大的等宽粗体数字，优先保证 1–3 位剩余百分比清晰可读，不在右侧重复显示。
3. **C · 分段电池**：macOS 风格电池轮廓，内部显示五个分段，百分比显示在电池右侧。

左侧 `Codex` 文字默认显示。关闭后只显示电池和百分比；样式 B 关闭文字后只显示数字徽章。

样式图形使用 AppKit 矢量绘制，并作为 `NSStatusItem` button 的模板图像显示，以适配 macOS 浅色、深色和菜单栏高亮状态，不使用 Unicode 方块字符模拟。

## 交互

左键点击菜单栏项目打开菜单，不使用单击循环切换。

菜单顺序：

1. 自定义信息区：第一行显示 `更新时间：HH:mm`；第二行显示 `下次重置：X 天 X 小时`；
2. 分隔线；
3. `A · 原生电池`、`B · 数字内嵌`、`C · 分段电池`，当前样式显示勾选；
4. `显示 Codex 文字`，开启时显示勾选；
5. 分隔线；
6. `退出`。

选择样式或切换文字显示后立即更新菜单栏，并更新菜单勾选状态。

三条样式菜单项使用自定义视图：左侧显示勾选状态和名称，右侧显示该样式的真实矢量电池示例图。点击任一样式后，顶部菜单栏效果与菜单勾选立即切换，但菜单保持打开，便于连续比较 A/B/C 与文字显示组合。

App 运行且 Mac 处于唤醒状态时，每 15 秒自动扫描一次本地最新额度记录；从睡眠恢复后继续扫描。菜单不提供手动刷新按钮，避免让用户误以为只有点击才会更新。扫描只读取 Codex 已写入 `~/.codex/sessions` 的最新快照，不调用私有接口；额度是否变化取决于 Codex 是否写入了新快照。

## 配置持久化

使用 `UserDefaults.standard` 保存：

- `batteryStyle`：`native`、`embedded` 或 `segmented`；缺失或非法值回退到 `native`。
- `showsCodexLabel`：布尔值；首次运行默认 `true`。

配置只影响视觉，不影响额度来源、刷新频率或百分比计算。

## 数据与无数据状态

- 百分比继续表示剩余额度。
- 多个额度窗口继续显示剩余最少的窗口。
- A/C 的无数据状态显示空电池和 `--%`。
- B 的无数据状态在数字徽章内显示 `--`。
- 首行的 `更新时间` 显示最新额度快照的时间，使用系统当前时区并精确到秒，格式为 `HH:mm:ss`；无数据时显示 `更新时间：--:--:--`。
- `下次重置` 使用当前实际限制窗口返回的服务端 `resets_at` 计算相对当前时间的剩余整天和整小时，格式为 `X 天 X 小时`，例如 `2 天 3 小时`；已到期显示 `0 天 0 小时`，缺失时显示 `--`。
- 重置倒计时随每次 15 秒自动扫描更新，不把本地推算结果写回额度快照。

## 分发

- 继续构建纯 Apple Silicon `arm64` App，兼容 M1–M5。
- 最低系统版本维持 macOS 13。
- 输出目录同时包含：
  - `Codex Quota.app`
  - `Codex Quota-arm64.zip`
- ZIP 使用 macOS `ditto -c -k --sequesterRsrc --keepParent` 创建，保留 bundle 元数据。
- 构建脚本在 ZIP 创建前验证测试、App 结构、图标、权限和签名；失败时保留上一份可用 App 和 ZIP。
- 当前继续使用 ad-hoc 签名。同事首次打开可能需要右键 App 后选择“打开”；完全消除 Gatekeeper 提示不在本次范围，需要 Apple Developer ID 签名和公证。
- 同事机器必须已经使用 Codex，并存在 `~/.codex/sessions` 本地额度记录；否则 App 显示无数据状态。

## 测试与验收

1. 三种样式在 `0%`、`49%`、`100%` 和无数据状态下生成正确尺寸的矢量菜单栏图像。
2. A 为默认样式；非法持久化值回退 A。
3. `Codex` 文字默认显示，并可切换、持久化和恢复。
4. 菜单勾选状态与实际渲染保持一致。
5. 切换设置不触发额度重新计算或文件扫描。
6. 浅色和深色菜单栏下图形可辨认，文字与电池垂直居中，无裁切。
7. 现有额度解析、扫描、性能和生命周期测试继续通过。
8. App bundle 验证、arm64 架构、ad-hoc 签名、Launch Services 注册继续通过。
9. ZIP 可解压为有效 App，解压后的签名和 bundle 类型验证通过。
10. 实际启动后可通过左键菜单切换三种样式和文字显示；退出后无残留进程。
11. 菜单中不显示手动刷新按钮；App 运行且 Mac 唤醒时自动刷新间隔仍为 15 秒。
12. 三条样式行右侧各显示对应矢量示例；点击切换时菜单保持打开，可连续组合样式与 `Codex` 文字开关。
13. 解析器保留所选限制窗口的 `resets_at`，重置倒计时覆盖未来、已到期和缺失三种状态。
14. B 的 1 位、2 位和 3 位数字在菜单栏与菜单预览中均可辨认，且不受电池外框挤压。
15. 每个用户首次启动 App 时显示一次原生提示框：标题为 `Codex Quota 已启动`，内容为 `额度每 15 秒自动更新一次，无需手动刷新。`，按钮为 `知道了`；关闭后持久化，后续启动不再显示。

## 明确不做

- 不改为 Intel 或 universal binary。
- 不增加开机自启、通知、颜色设置或独立设置窗口。
- 不调用私有 OpenAI 接口。
- 不申请或配置 Apple Developer ID、公证或 Mac App Store 分发。
